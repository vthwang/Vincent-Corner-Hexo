Nodejs II 快速學習自我挑戰 Day2

2017-11-10
Nodejs
Symbols count in article: 1.9k Reading time: 10 mins.

Mongoose 和本地端 MongoDB

安裝 Mongoose 和連接

  1. 安裝 Mongoose npm install --save mongoose
  2. Mongoose 官方網站
  3. 在 app.js 新增 mongoose connect 連接到本地端伺服器，記得要使用 useMongoClient，不然會出現奇怪的錯誤，如果連接成功，顯示 MongoDB connected，失敗則出現錯誤
  1. 加入這一行避免 deprecation 警告出現，把預設已經 deprecated 的 library 用 global promise 取代
    mongoose.Promise = global.Promise;

建立理想的 Model

  1. 新增 models/Idea.js，新增 ideas 的 model
  1. 在 app.js 讀取 models

新增 Idea Form

  1. 在 views/partials/_navbar.handlebars 加入右側選單，直接加在左側選單的 ul 下面
  1. 在 app.js 新增 ideas/add 的路由
  1. 新增 views/ideas/add.handlebars

伺服端的 Form Validation

  1. Body-Parser 外掛
  2. 安裝 body-parser npm install body-parser --save
  3. 在 app.js 引入 body-parser const bodyParser = require('body-parser');
  4. 在 app.js 呼叫 Body parser middleware
  1. Server Side 處理表單，如果沒有資料，回傳 text，並且將本來有輸入的值放回去，最後，如果全部都有資料則 passed
  1. 在 views/ideas/add.handlebars 新增錯誤訊息，將剛剛回傳的 errors 放進來
  1. 在 views/ideas/add.handlebars 的表單輸入加入 required，可使用前端認證

將 Idea 存到 MongoDB

  1. 在 app.js 確認沒有錯誤的話則寫入資料庫
  1. 進入 mongo 模式 mongo
  2. 查詢資料庫
    • 顯示資料庫 show dbs
    • 進入資料庫 use vidjot-dev
    • 顯示資料庫內的 collections show collections
    • 讀取指定 collection 的內容 db.ideas.find();

從 MongoDB 讀取 Ideas

  1. 在 app.js 新增 ideas 的路由，然後傳送資料過去，且排列順序按照 date 降冪排列
  1. 新增 views/ideas/index.handlebars 並用 each 讀出資料

編輯 Idea Form

  1. 在 app.js 新增編輯 Idea Form 的路由，只找尋一個然後傳過去下個頁面
  1. 在 views/ideas/index.handlebars 新增一個 Edit 的連結
    <a class="btn btn-dark btn-block" href="/ideas/edit/{{id}}">Edit</a>
  2. 從 views/ideas/add.handlebars 複製新增到 views/ideas/edit.handlebars，且將值放進去

更新 Idea Data

  1. Method Override
  2. 安裝 method-override npm install method-override --save
  3. 在 app.js 引入 const methodOverride = require('method-override');
  4. 在 app.js 新增 method-override 的 middleware app.use(methodOverride('_method'));
  5. 修改 views/ideas/edit.handlebars 的表單 action 且新增 hidden input
  1. 在 app.js 更新表單，然後將值存到資料庫

移除 Idea Data

  1. 在 views/ideas/index.handlebars 新增 Delete 按鈕
  1. 在 app.js 新增刪除路由，用 remove 直接把檔案從資料庫刪除

Flashing Messaging

  1. Express Session
  2. Connect Flash
  3. 安裝 Express Session 和 Connect Flash npm install express-session connect-flash --save
  4. 在 app.js 引入 Express Session 和 Connect Flash
  1. 在 app.js 把 Express Session 和 Connect Flash 的 middleware
  1. 在 app.js 把區域變數變成全域變數，這樣訊息就可以在 handlebars 的頁面顯示
  1. app.js 在所有路由跳轉之前，送出 flash 訊息
    req.flash('success_msg', 'Video Idea added');
    req.flash('success_msg', 'Video Idea updated');
    req.flash('success_msg', 'Video Idea removed');
  2. 新增 views/partials/_msg.handlebars
  1. 在 views/layouts/main.handlebars 的 {{"{{body"}}}} 上方引入 {{"{{> _msg"}}}}
