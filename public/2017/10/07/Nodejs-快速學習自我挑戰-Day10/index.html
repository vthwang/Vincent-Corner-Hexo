Nodejs 快速學習自我挑戰 Day10

2017-10-08
Nodejs
Symbols count in article: 1.7k Reading time: 9 mins.

安全和認證

讓 Todo Routes Private

  1. 修改 server/models/todo.js 新增 _creator
  1. 修改 server/tests/seed/seed.js 新增 _creator
  1. 修改 server/server.js，加入 authenticate，然後新增 _creator
  1. 在 server/tests/server.test.js 的 POST 部分設置 x-auth
    .set('x-auth', users[0].tokens[0].token)
  2. 在 server/tests/server.test.js 的 GET 部分設置 x-auth，並在回送時設定長度為 1，因為設置了 creator，只會回傳一個 todo
  1. 在 server/server.js 修改 POST /users/login 的 token 為 1
  1. 在 server/server.js 修改 GET /todos/:id’，新增 authenticate middleware，且修改 findById 成 findOne
  1. 在 server/tests/server.test.js 修改 GET /todos 和 GET /todo/:id 都加上 .set('x-auth', users[0].tokens[0].token)
  2. 在 server/server.js 修改 DELETE /todos/:id’，新增 authenticate middleware，且修改 findByIdAndRemove 成 findOneAndRemove
  1. 在 server/tests/server.test.js 的 describe(‘GET /todo/:id’) 加上一個測試，如果被其他使用者建立的 todo 則 404
  1. 在 server/tests/server.test.js 的 DELETE /todos/:id 的每個測試都加上
    .set('x-auth', users[1].tokens[0].token)
  2. 在新增一個測試，如果 todo 對不上 user 就回傳 404
  1. 在 server/server.js 修改 PATCH /todos/:id’，新增 authenticate middleware，且修改 findByIdAndUpdate 成 findOneAndUpdate

改善 App 組態設定

  1. 在 server/config/config.js 修改成用 json 的方式讀取環境變數
  1. 新增 server/config/config.json，把組態檔放進去
  1. 在 server/tests/seed/seed.js 的 jwt.sign 值都改為 process.env.JWT_SECRET
  2. 在 server/tests/seed/user.js 的 jwt.sign 和 jwt.verify 值都改為 process.env.JWT_SECRET
  3. 在 heroku 設定 JWT_SECRET
    heroku config:set JWT_SECRET=fadskljfa3kfwe345mvvs2g3eio3

部署到 Heroku

  1. 取得 MONGODB_URI
    heroku config:get MONGODB_URI
  2. 取得值 - mongodb://帳號:密碼@連線網址:39994/資料庫名稱
  3. 進入 Robo 3T 進行新的連線並測試
  4. git push master heroku

進階的 Postman

  1. 在 POST /users 和 /users/login 的 Test 選項寫入下面兩行，在送出時就會設定 x-auth 為全域的變數
  1. 在其他的 header 加上 {{x-auth}} 取得變數
  2. 在 POST /todos 的 Test 選項寫入下面兩行，在送出時就會設定 todoId 為全域的變數
  1. 在其他有 id 的 url 上面加上 {{todoId}}

升級 expect

  1. npm install expect@21.2.1 --save-dev
  2. 把 server/tests/server.test.js 的 toNotExist 改成 toBeFalsy，把 toExist 改成 toBeTruthy
  3. 在 PATCH /todos/:id 的部份把 expect(res.body.todo.completedAt).toBeA('number'); 取代為 expect(typeof res.body.todo.completedAt).toBe('number');
  4. 將 .toNotBe 改為 .not.toBe
  5. 將 toInclude 改為 toMatchObject，toObject 會拋回原始的 user 資料
    expect(user.toObject().tokens[1]).toMatchObject({

使用 Socket.io 的即時 Web Apps

創建一個專案

  1. mkdir node-chat-app
  2. cd node-chat-app
  3. npm init
  4. git init
  5. 新增 server/server.js
  1. 新增 public/index.html
  1. npm i express --save
  2. 推到 Github
  1. 發佈到 heroku

增加 Socket.io 到 App

  1. npm i socket.io --save
  2. 新增 http 和 socketIO 到 server/server.js，
  1. 新增 socket.io 到 index.html

Events 的 Emitting 和 Listening

  1. 新增 public/js/index.js
  1. 在 public/index.html 引入 script
    <script src="js/index.js"></script>
  2. 在 server/server.js 在 connect 的時候，傳出 newEmail，然後等待接收 createEmail
  1. 在 public/js/index.js 新增 emit
  1. 在 server/server.js 接收 client 的 emit
  1. 在 server/server.js 新增 emit
  1. 在 public/js/index.js 把接收的 Message console 出來
