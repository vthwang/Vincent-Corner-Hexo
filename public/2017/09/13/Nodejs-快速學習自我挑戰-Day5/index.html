Nodejs 快速學習自我挑戰 Day5

2017-09-14
Nodejs
Symbols count in article: 2.3k Reading time: 13 mins.

網頁伺服器及應用程式部署

進階 template

  1. 註冊 partials
    hbs.registerPartials(__dirname + '/views/partials')
  2. 將 footer code 移到 views/partials/footer.hbs
  1. 本來 footer 的區塊用 {{" {{> footer" }}}} 取代
  2. nodemon 監控 js 和 hbs 檔案 nodemon server.js -e js, hbs
  3. 將 header code 移到 views/partials/header.hbs
  1. 本來 footer 的區塊用 {{" {{> header" }}}} 取代
  2. 使用 hbs 的 Helper 來取得年份
  1. 本來 {{" {{currentYear"}}}} 用 {{" {{getCurrentYear" }}}} 取代
  2. 使用 hbs 的 Helper 註冊 screamIt 然後將 text 變數送進去
  1. 使用 screamIt，text 變數設為 welcomeMessage{{" {{screamIt welcomeMessage" }}}}

Express Middleware

  1. 註冊一個 middleware，然後 console 時間以及擷取的狀態
  1. 新增 maintenance.hbs
  1. 使用 middleware 把頁面通通導向 maintenance

Git 版本控制

  1. Git 官方網站
  2. 啟動 git 專案 git init
  3. 新增檔案 git add
  4. 新增 .gitignore 讓部分檔案不上傳
  5. 對新增的檔案做 commit git commit -m 'Commit Content'

設置 ssh key

  1. ssh key 目錄 ~/.ssh
  2. 產生 ssh key ssh-keygen -t rsa -b 4096 -C 'YOUR@EMAIL'
  3. 設定完成會產生出 id_rsa(此為私密金鑰，不能給別人) 和 id_rsa_pub(此為公開金鑰，用於第三方軟件)
  4. 開啟 ssh-agent eval "$(ssh-agent -s)"
  5. 將 ssh 私鑰加入 ssh-agent ssh-add ~/.ssh/id_rsa
  6. 將 ssh key 貼到 Github => Setting => SSH and GPG key pbcopy < ~/.ssh/id_rsa.pub
  7. 跟 Github 進行連線 ssh -T git@github.com

部署 apps

  1. heroku
  2. 安裝 heroku cli brew install heroku
  3. 在本地端 cli 登入 heroku heroku login
  4. 新增 ssh key 到 keroku heroku keys:add
  5. 檢查在電腦上的 key heroku keys
  6. 跟 heroku 進行連線 ssh -v git@heroku.com
  7. 修改 server.js 讓 port 動態調整，先取得 env 的 PORT，如果不存在則使用預設的 3000 Port
    const port = process.env.PORT || 3000;
  8. 修改 server.js 的 app.listen
  1. 在 package.json 的 script 新增 start，讓 server 知道要執行哪個檔案
  1. 開啟 heroku 專案 heroku create
  2. 將專案推到 heroku git push heroku
  3. 用瀏覽器開啟 keroku 的專案heroku open

測試 Apps

Mocha 和基本測試

  1. mkdir node-tests
  2. cd node-tests
  3. npm init，全部直接使用預設值
  4. 新增檔案 utils/utils.js
    module.exports.add = (a, b) => a + b;
  5. Mocha 官方網站
  6. 安裝 mocha 且不在正式上線時使用，僅用於開發用途
    npm i mocha@3.0.0 --save-dev
  7. 新增 utils/util.test.js
  1. 修改 package.json，**搜尋所有資料夾
  1. npm test 執行測試
  2. 加一行錯誤
    throw new Error('Value not correct!')
  3. 再執行 npm test，就會發現錯誤了
  4. 加 if statement，回傳得到的錯誤值
  1. 在 utils/utils.js 新增一個 module
    module.exports.square = (x) => x * x;
  2. 在 utils.utils.test.js 多新增一個 test

監控和自動重啟測試

  1. nodemon --exec 'npm test'
  2. 在 package.json 直接新增 command
    "test-watch": "nodemon --exec \"npm test\""
  3. npm run test-watch

使用 Assertion Library

  1. Assertion Library
  2. 安裝 expect npm install expect@1.20.2 --save-dev
  3. 用 expect 簡化函數
  1. 在 utils/utils.js 新增 module
  1. 用 expect 確認回傳的物件是正確值

測試非同步式程式碼

  1. 在 utils/utils.js 新增一個 asyncAdd
  1. 在 utils/utils.test.js 測試 asyncAdd，這邊一定要使用 done，這樣才知道他是非同步，會等完成再執行 expect
  1. 在 utils/utils.js 新增一個 asyncSquare
  1. 在 utils/utils.test.js 測試 asyncSqure

測試 Express 應用

  1. 安裝 Express
    npm i express@4.14.0 --save
  2. 新增 server/server.js
  1. 執行 server
    node server/server.js
  2. 去瀏覽器輸入 http://localhost:3000/ 就會看到 Hello world!
  3. SuperTest
  4. npm i supertest@2.0.0 --save-dev
  5. 修改 server/server.js 輸出內容讓其它地方可用
    module.exports.app = app;
  6. 新增 server/server.test.js
  1. 引入 expect，用 expect 客製化測試
  1. 在 server/server.js 新增路由
  1. 在 server/server.test.js 新增測試

使用 describe() 組織測試

  1. 修改 utils/utils.test.js
  1. 修改 server/server.test.js

Test Spies

  1. 新增 spies/db.js
  1. 新增 spies/app.js
  1. 新增 spies/app.test.js
  1. npm install rewire@2.5.2 --save-dev
  2. 在 spies/app.test.js 使用 rewire 引入 app
    var app = rewire('./app');
  3. 用 describe 將 rewire 單元測試分類在一起
